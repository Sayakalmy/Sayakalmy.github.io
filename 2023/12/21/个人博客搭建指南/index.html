个人博客搭建指南

一、安装 Node.js

在Node.js官网进行下载https://nodejs.org/en/download

选择左边的稳定版进行下载，即（Recommend For Most Users），安装完成后可以在命令行中输入node -v，若能正确显示版本号，则说明安装成功。

二、安装git

在git官网进行下载https://git-scm.com/downloads

安装完成后在命令行中输入git -v，若能正确显示版本号，则说明安装成功。

三、安装Hexo

Hexo 是一个基于 Node.js 开发的静态网页生成器，官网的说明是博客框架，但是我认为它不仅仅可以用于博客，还可以用于企业宣传站、产品展示、文本文档等以信息展示为主的网站。

首先在本地的位置新建一个安装博客的文件夹，在文件夹目录右键选择Git Bash Here。

接着在git中以此输入以下命令：

npm install hexo-cli -g

hexo init blog

cd blog

npm install

hexo server

四、创建GitHub仓库

进入GitHub官网（需要魔法上网）

如果之前有账号了，就可以直接登录（相信大家肯定都有吧），如果没有就点击Sign Up进行注册，按照要求完成注册信息。

创建完成后，需要设置 ssh key（如果已经设置过就不需要了）。这个key可以理解为一个身份密码，你可以使用指令生成一个公钥和私钥，只有当你把你这台设备的公钥上传到GitHub上，GitHub才能和你的本地仓库建立联系。

如果你的git还没有进行全局设置，可以先进行全局设置：

git config --global user.name "你的用户名"

git config --global user.email "你的邮箱"

接下来在任意目录中打开git，输入命令cd ~/.ssh，将工作路径切换到隐藏目录.ssh中。

然后输入命令ssh-keygen -t rsa -C "your email"生成密钥。注意把your email替换成你的邮箱。

此时可以用ls命令来查看当前目录的内容，若出现id_rsa和id_rsa.pub，则配置成功。

接着输入cat id_rsa.pub来查看id_rsa.pub文件中的信息，并将其复制。

接下来登录GitHub，点击右上角你的头像，选择Settings，

点击左边列表中SSH and GDP keys，点击右上角的New SSH key进入添加密钥界面。

在Title处添加密钥的名字，然后在Key中黏贴刚刚复制的密钥，这样就可以在GitHub上添加这个公钥，在上一级页面中可以看到你的SSH公钥。

然后在git中输入ssh -T git@github.com，若返回successfully则说明配置成功。

然后点击GitHub页面右上角的头像，点击your profile进入你的个人主页

点击右上角加号，在里面点击New repository新建一个仓库.

在repository name中输入仓库名称。这里仓库名称必须是你的用户名+“.github.io”，不然会出问题。description中输入你想要的描述（可以不写），然后点击Create Repository新建仓库。

五、上传本地博客至GitHub仓库

来到个人博客blog根目录，打开里面的_config.yml文件，滑到最底部，修改里面的deploy项。(在修改前建议先复制一份作为副本)

在type中输入’git’，然后回车，新增加一个repository项，输入英文冒号和一个空格（注意：这个空格必须加，不然会报错）

然后来到你的GitHub新建的仓库页面，点击绿色code，在clone中切换到SSH，点击右边的两个长方形图标复制里面的内容

然后粘贴到刚刚repository冒号和空格后边，保存。这样子它就知道需要在GitHub的哪个仓库中存放你的博客了。如果你想要换一个仓库，只需要将这里的地址修改掉，然后重新执行以下步骤即可。

然后你需要再次按下回车，添加新的branch属性，同样需要添加冒号和空格，然后后面输入main（表示GitHub上的主分支）

之后在blog根目录中git bash，运行以下代码：

npm install hexo-deployer-git --save

接下来就可以使用hexo g生成网页内容并使用hexo s预览博客，当然也可以直接输入hexo d部署博客。

这个时候我们可以回到刚刚创建的仓库，在main分支上就可以看到刚刚上传的代码。（如果你的代码上传到了其他分支上说明你错了，这样会出bug）

接下来你就可以点击页面右上方的settings，左边点击pages，就可以看到你部署的仓库的网页网址了。
